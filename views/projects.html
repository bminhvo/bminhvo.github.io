Software Engineering Class - Group Project

Teal The Social Application

My Responsibility: Front-end Developer

March - May 2017

This web application functions as a social connector and its primary purpose is to facilitate the connection of users who have some certain needs with other users who can fulfil those needs. Hence, users can search for, connect with, and make their requests to other users who may not be within close geographic proximity.

Teal has some features integrated from popular social networking sites such as Facebook and Twitter. However, it also has certain features that are unique to it. The bulk of the application's framework was built using Rails 4.2.5. PostgreSQL 9.3.15 served as the database system and Bootstrap was used to design the front-end UI.

Within the group of five members, we decided to divide the work up so that we could finish the project efficiently. I was assigned the role of a Front-end developer. From the prototype/template that we got, my job was to alter and improve the elements of the UI as well as the UX according to our group's planning and the users' feedback. I also partially contributed to the Back-end development of the web application during our pair programming sessions.

Software Engineering Class - Group Project

Amazon - Nike Compare Web Application

My Responsibility: Back-end Developer

January - February 2017

Given Nike product names, this web application collects the prices (among other pieces of data) from amazon.com and nike.com, analyzes it, and presents it for the user to compare the prices on the two websites. The average prices are calculated and visualized as a bar chart. Users can also search the results using specific details such as product descriptions, prices or ratings.

The application itself was built with AngularJS and the user interface was designed with Twitter’s Bootstrap. Angular Material was conveniently used to build most of the UI features. D3.js is used to draw the bar chart using the average prices from Nike.com every time a product is searched.

Python modules were used to build the server, scrape the data, and perform the data analysis. The server was implemented with Flask and BeautifulSoup was used to scrape data from amazon.com and nike.com. Given a keyword, the bs4 makes search requests to amazon.com and nike.com by passing the keyword as a query string in the URL. It then downloads the returned HTML files and selects from the HTML tree the required HTML elements with certain classes and/or attributes. The selected relevant data is then saved into a JSON file to be used as the model in Angular’s MVC.

My role in the group of three members was to implemente Flask and BeautifulSoup to scrape data from amazon.com and nike.com based on user’s searches. To go into more details, I had to examined the HTML tree and designed a way to select certain tags/attributes to extract data about the products, which were saved into a JSON file for other members to use for displaying purposes.
